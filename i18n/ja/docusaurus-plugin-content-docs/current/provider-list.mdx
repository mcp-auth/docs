# MCP 互換プロバイダー一覧

この一覧には、MCP Auth で動作確認済みのプロバイダーが含まれています。

| プロバイダー                                              | 種類            | OAuth 2.1 | メタデータ URL | 動的クライアント登録 | リソースインジケーター[^1] |
| --------------------------------------------------------- | --------------- | --------- | -------------- | --------------------- | -------------------------- |
| [Logto](https://logto.io)                                 | OpenID Connect  | ✅        | ✅             | ❌[^2]                | ✅                         |
| [Keycloak](https://www.keycloak.org)                      | OpenID Connect  | ✅        | ✅             | ⚠️[^3]                | ❌                         |
| [Auth0](https://www.auth0.com)                            | OpenID Connect  | ✅        | ✅             | ✅                    | ⚠️[^4]                     |
| [Asgardeo](https://wso2.com/asgardeo)                     | OpenID Connect  | ✅        | ✅             | ✅                    | ❌                         |
| [WSO2 Identity Server](https://wso2.com/identity-server/) | OpenID Connect  | ✅        | ✅             | ✅                    | ❌                         |

他のプロバイダーで MCP Auth の動作確認ができた場合は、ぜひプルリクエストでこの一覧に追加してください。

[^1]: リソースインジケーターは [RFC 8707: OAuth 2.0 のリソースインジケーター](https://datatracker.ietf.org/doc/html/rfc8707) を指し、クライアントがアクセスしたいリソースを示すための標準です。

[^2]: Logto は動的クライアント登録のサポートを追加予定です。

[^3]: Keycloak は動的クライアント登録をサポートしていますが、クライアント登録エンドポイントが CORS をサポートしていないため、ほとんどの MCP クライアントが直接登録できません。

[^4]: Auth0 はマルチリソースリフレッシュトークン (MRRT) をサポートしていますが、RFC 8707 の完全なサポートではありません。リソースインジケーターのサポートは限定的で標準準拠ではありません。

## 動的クライアント登録は必須ですか？ \{#is-dcr-required}

[動的クライアント登録](https://datatracker.ietf.org/doc/html/rfc7591) は MCP サーバーや MCP Auth で必須ではありません。実際には、ニーズに最適な方法を選択できます：

1. **社内利用や自分が管理する特定アプリケーション向けに MCP サーバーを開発している場合**：プロバイダーに MCP クライアントを手動で登録し、クライアント ID（必要に応じてクライアントシークレットも）を MCP クライアントに設定すれば問題ありません。
2. **公開アプリケーション（MCP クライアント）で利用される MCP サーバーを開発している場合**：
   1. 動的クライアント登録を活用し、MCP クライアントがプロバイダーに自動登録できるようにできます。不正または悪意のある登録を防ぐため、適切なセキュリティ対策を実装してください。
   2. あるいは、独自の登録フローを開発し、MCP クライアントが安全かつ管理されたプロセス（Web インターフェースや自分が管理する API エンドポイントなど）でプロバイダーに登録できるようにすることも可能です。動的クライアント登録に依存する必要はありません。
      プロバイダーが Management API や同等の機能をサポートしていれば、独自エンドポイントで MCP クライアントの登録に利用できます。
