# MCP 互換プロバイダーリスト

このリストには、MCP Auth でテスト済みのプロバイダーが含まれています。

| プロバイダー                                                  | 種類             | OAuth 2.1 | メタデータ URL | 動的クライアント登録 | リソースインジケーター[^1] |
| ------------------------------------------------------------- | ---------------- | --------- | -------------- | --------------------- | -------------------------- |
| [Logto](https://logto.io)                                     | OpenID Connect   | ✅        | ✅             | ❌[^2]                | ✅                         |
| [Keycloak](https://www.keycloak.org)                          | OpenID Connect   | ✅        | ✅             | ⚠️[^3]                | ❌                         |
| [Asgardeo](https://wso2.com/asgardeo)                         | OpenID Connect   | ✅        | ✅             | ✅                    | ❌                         |
| [WSO2 Identity Server](https://wso2.com/identity-server/)     | OpenID Connect   | ✅        | ✅             | ✅                    | ❌                         |
| [Auth0](https://www.auth0.com)                                | OpenID Connect   | ✅        | ✅             | ✅                    | ⚠️[^4]                    |
| [Descope](https://www.descope.com)                            | OpenID Connect   | ✅        | ✅             | ✅                    | ⚠️[^4]                    |

他のプロバイダーで MCP Auth の動作確認ができた場合は、ぜひプルリクエストでリストへの追加をお願いします。

[^1]: リソースインジケーターは [RFC 8707: OAuth 2.0 のリソースインジケーター](https://datatracker.ietf.org/doc/html/rfc8707) を指し、クライアントがアクセスしたいリソースを示す標準です。

[^2]: Logto は動的クライアント登録のサポートを追加予定です。

[^3]: Keycloak は動的クライアント登録をサポートしていますが、クライアント登録エンドポイントが CORS をサポートしていないため、多くの MCP クライアントが直接登録できません。

[^4]: Auth0 および Descope はマルチリソースリフレッシュトークン (MRRT) をサポートしていますが、RFC 8707 の完全なサポートではありません。リソースインジケーターのサポートは限定的で標準準拠ではありません。

## 動的クライアント登録は必須ですか？ \{#is-dcr-required}

[動的クライアント登録](https://datatracker.ietf.org/doc/html/rfc7591) は MCP サーバーや MCP Auth で必須ではありません。実際、ニーズに最適な方法を選択できます：

1. **社内利用や特定のアプリケーション向けに MCP サーバーを開発している場合**：プロバイダーに MCP クライアントを手動で登録し、クライアント ID（および必要に応じてクライアントシークレット）を MCP クライアントに設定しても問題ありません。
2. **公開アプリケーション（MCP クライアント）で利用される MCP サーバーを開発している場合**：
   1. 動的クライアント登録を活用し、MCP クライアントがプロバイダーに自動登録できるようにできます。この場合、不正または悪意のある登録を防ぐために適切なセキュリティ対策を実装してください。
   2. あるいは、動的クライアント登録に依存せず、Web インターフェースや管理 API エンドポイントなど、安全かつ管理されたプロセスで MCP クライアントをプロバイダーに登録できる独自の登録フローを開発することも可能です。
      プロバイダーが Management API や同等の機能をサポートしていれば、独自エンドポイントで MCP クライアントの登録に利用できます。
